# ADR 003：来源锚点格式设计与防幻觉机制

**日期：** 2026-03-14
**状态：** 已确认

## 决策

报告中所有事实性声明必须在**同一句或紧跟的下一行**附带来源锚点。
锚点格式设计为机器可解析，便于 `verify_citations.py` 自动核查。

## 锚点格式规范

```
[Z: ItemKey | 作者 年份 | 标题片段]        ← Zotero 库内项目（最高置信度）
[A: doi:10.xxx | 作者 年份 | 期刊名]       ← 学术数据库论文
[W: URL | 页面标题 | 访问日期YYYY-MM-DD]   ← 网络来源
[L: D:/完整路径/file.pdf | p.N]            ← 本地文件（含页码）
```

## 执行机制

`verify_citations.py` 使用正则解析这四种锚点：
- `[Z: ...]` → sqlite3 查询 zotero.sqlite，确认 ItemKey 存在
- `[A: doi:...]` → Semantic Scholar API 确认 DOI 有效
- `[W: ...]` → HTTP HEAD 请求确认 URL 可访问
- `[L: ...]` → Path.exists() 确认文件存在

**长段落无锚点检测：** 超过 100 字符的正文段落若无紧跟锚点，标记为"潜在未引用声明"。

## 置信等级与锚点要求

| 等级 | 锚点要求 |
|------|---------|
| High | 至少 1 个 [Z:] + 1 个 [A:] 或另一个 [Z:] |
| Medium | 至少 1 个 [A:] 或 [W:] |
| Low | 至少 1 个任意锚点 |
| UNVERIFIED | 无锚点 → 必须移出正文 |
